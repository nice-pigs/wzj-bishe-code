# ESP8266与STM32连接说明

## 硬件连接

### 使用软件串口连接（推荐）
```
ESP8266 NodeMCU    →    STM32F103C8T6
D1 (GPIO5/TX)      →    PB11 (USART3 RX)
D2 (GPIO4/RX)      →    PB10 (USART3 TX)
3V3                →    3.3V
GND                →    GND
```

**注意：这是交叉连接！**
- ESP8266的TX(D1) 连接 STM32的RX(PB11)
- ESP8266的RX(D2) 连接 STM32的TX(PB10)

## 引脚位置

### ESP32开发板引脚布局
```
                ESP32开发板
    ┌─────────────────────────────────┐
    │ VIN  GEN  RST  EN   3V3  GND   │
    │                                 │
    │ SDO  CMD  SD1  SD2  SD3  D0    │
    │                                 │
    │ D1   D3   D4   D5   D6   D7    │
    │                                 │
    │ RX   TX                         │  ← 使用这两个引脚
    └─────────────────────────────────┘
```

### STM32F103C8T6引脚
- **PB10**: USART3的TX引脚
- **PB11**: USART3的RX引脚
- **3.3V**: 电源正极
- **GND**: 电源负极

## 连接步骤

1. **断电连接**：确保两个板子都断电
2. **连接信号线**：
   - ESP32的RX → STM32的PB10
   - ESP32的TX → STM32的PB11
3. **连接电源线**：
   - ESP32的3V3 → STM32的3.3V
   - ESP32的GND → STM32的GND
4. **检查连接**：确保连接牢固，无短路
5. **上电测试**：先给STM32上电，再给ESP32上电

## 注意事项

⚠️ **重要提醒**：
- ESP32的RX连接STM32的TX(PB10)
- ESP32的TX连接STM32的RX(PB11)
- 这是交叉连接，不要接错！
- 电源电压必须是3.3V，不能用5V
- 连接前务必断电，避免损坏芯片

## 测试方法

1. **上传STM32程序**：编译并下载STM32代码
2. **上传ESP32程序**：修改WiFi配置后上传ESP32代码
3. **查看串口输出**：
   - STM32串口(PA9/PA10)：查看系统运行状态
   - ESP32 USB串口：查看WiFi连接状态和IP地址
4. **访问Web界面**：在浏览器中输入ESP32的IP地址

## 故障排除

### 连接问题
- 检查线路是否接错（特别是RX/TX交叉连接）
- 确认电源电压是3.3V
- 检查GND是否连接

### 通信问题
- 确认波特率都是115200
- 检查STM32的USART3是否正确初始化
- 查看ESP32串口监视器是否有数据接收

### WiFi问题
- 确认WiFi名称和密码正确
- 检查WiFi信号强度
- 尝试重启ESP32
