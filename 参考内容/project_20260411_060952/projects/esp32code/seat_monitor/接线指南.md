## 详细接线

### 1. HC-SR501 红外传感器

| HC-SR501 | ESP32-S3 | 说明 |
|----------|----------|------|
| VCC | 5V 或 3.3V | 电源（推荐5V） |
| GND | GND | 地线 |
| OUT | GPIO7 | 信号输出 |

**HC-SR501 调节说明：**
- **灵敏度**：顺时针旋转增加检测距离（最大7米）
- **延时**：顺时针旋转增加延时（最大约5分钟），建议调到最小
- **跳线选择**：选择 H（可重复触发模式）

### 2. OLED 显示屏 (SSD1306 I2C)

| OLED | ESP32-S3 | 说明 |
|------|----------|------|
| VCC | 3.3V 或 5V | 电源 |
| GND | GND | 地线 |
| SCL | GPIO5 | I2C 时钟 |
| SDA | GPIO6 | I2C 数据 |

**I2C 地址**：默认 0x3C（如果显示不亮，尝试 0x3D）

### 3. LED 指示灯

**红色 LED（有人时亮）：**
```
ESP32 GPIO8 ──[220Ω电阻]──LED(+)──LED(-)──GND
```

**绿色 LED（无人时亮）：**
```
ESP32 GPIO9 ──[220Ω电阻]──LED(+)──LED(-)──GND
```

> **注意**：LED 长脚是正极(+)，短脚是负极(-)

## 引脚定义表

```cpp
#define PIR_PIN        7      // HC-SR501 信号引脚
#define LED_RED_PIN    8      // 红色LED (有人)
#define LED_GREEN_PIN  9      // 绿色LED (无人)
#define SDA_PIN        6      // OLED I2C SDA
#define SCL_PIN        5      // OLED I2C SCL
```

> 如果你的 ESP32-S3 开发板引脚不同，请在代码中修改这些定义

## 常见问题

### 1. OLED 不显示
- 检查 I2C 地址（尝试 0x3C 或 0x3D）
- 检查接线是否正确
- 检查电源是否接好

### 2. HC-SR501 一直检测到人
- 调低灵敏度
- 检查延时设置（调到最小）
- 避免对着热源或阳光

### 3. HC-SR501 检测不到人
- 调高灵敏度
- 检查供电电压（5V 更稳定）
- 检查 OUT 引脚是否接对

### 4. WiFi 连接失败
- 检查 WiFi 名称和密码
- 确认 WiFi 是 2.4GHz（ESP32 不支持 5GHz）
- 检查信号强度

## 电源供电

推荐供电方式：
- USB-C 供电（最方便）
- 5V 电源适配器
- 移动电源

功耗估算：
- 正常工作：约 100-150mA
- WiFi 发送时：约 200-300mA
