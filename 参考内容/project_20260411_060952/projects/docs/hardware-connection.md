# 硬件连接指南

## 接线总览

```
                    STM32F103C8T6 最小系统板
                    ┌─────────────────────┐
                    │                     │
    红外传感器 ────→│ PA1        PB6  ───→│ OLED SCL
                    │                     │
    ESP8266 RX ────→│ PA9 (TX)   PB7  ───→│ OLED SDA
    ESP8266 TX ────→│ PA10 (RX)  PB1  ───→│ 红色LED
                    │                     │
    ESP8266 RST ───→│ PB0        PB2  ───→│ 绿色LED
                    │                     │
        3.3V ──────→│ 3.3V       GND  ───→│ GND (共地)
        GND  ──────→│ GND              │
                    └─────────────────────┘
```

## 详细接线表

### 1. ESP8266 Wi-Fi模块

| ESP8266引脚 | STM32引脚 | 说明 |
|------------|----------|------|
| VCC | 3.3V | 供电（注意：必须3.3V） |
| GND | GND | 共地 |
| TX | PA10 (RX) | 串口通信 |
| RX | PA9 (TX) | 串口通信 |
| RST | PB0 | 复位控制 |
| CH_PD/EN | 3.3V | 使能引脚，必须接高电平 |

⚠️ **重要**：ESP8266 需要独立供电，电流峰值可达300mA！

### 2. OLED显示屏 (I2C接口)

| OLED引脚 | STM32引脚 | 说明 |
|---------|----------|------|
| VCC | 3.3V | 供电 |
| GND | GND | 共地 |
| SCL | PB6 | I2C时钟线 |
| SDA | PB7 | I2C数据线 |

### 3. 红外传感器 HC-SR501

| HC-SR501引脚 | STM32引脚 | 说明 |
|-------------|----------|------|
| VCC | 5V | 供电（5V） |
| GND | GND | 共地 |
| OUT | PA1 | 信号输出 |

**HC-SR501 调节说明：**
- 延时调节：逆时针拧到底（最小延时）
- 灵敏度调节：中间位置
- 跳线帽：选择 H（可重复触发）

### 4. LED指示灯

| LED | STM32引脚 | 串联电阻 |
|-----|----------|---------|
| 红色LED正极 | PB1 | 1KΩ |
| 绿色LED正极 | PB2 | 1KΩ |
| LED负极 | GND | - |

## 注意事项

### ⚠️ 电源问题（最容易出错！）

1. **ESP8266 必须稳定供电**
   - 使用独立的 3.3V 稳压模块
   - 并联 100μF 电容滤波
   - 不能直接用 STM32 的 3.3V 引脚供电

2. **共地**
   - 所有模块的 GND 必须连接在一起

3. **电压匹配**
   - STM32 和 ESP8266 都是 3.3V 电平
   - HC-SR501 是 5V 供电，但输出是 3.3V 兼容

### ⚠️ 引脚冲突

确保以下引脚没有被占用：
- PA9/PA10：用于串口1（Wi-Fi通信）
- PB6/PB7：用于 I2C1（OLED显示）
- PA13/PA14：SWD 调试接口（不要接线）

## 测试顺序

建议按以下顺序接线测试：

1. ✅ **先接 ST-Link**，下载一个简单程序测试
2. ✅ **接 LED**，测试 GPIO 输出
3. ✅ **接红外传感器**，测试输入检测
4. ✅ **接 OLED**，测试显示功能
5. ✅ **最后接 ESP8266**，测试 Wi-Fi 通信

这样可以逐步排查问题！
