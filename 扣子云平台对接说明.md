### 字段说明

| 字段 | 类型 | 说明 | 范围 |
|------|------|------|------|
| deviceKey | string | 设备唯一标识 | - |
| deviceName | string | 设备名称 | - |
| isOccupied | boolean | 座位占用状态 | true/false |
| temperature | number | 温度 | -40~80 °C |
| humidity | number | 湿度 | 0~100 % |
| lightLevel | number | 光照强度 | 0~65535 |
| pressure1 | number | 压力传感器1 | 0/1 |
| pressure2 | number | 压力传感器2 | 0/1 |
| distance | number | 超声波距离 | 0~400 cm |
| pirState | number | PIR红外状态 | 0/1 |
| wifiSignal | number | WiFi信号强度 | -100~0 dBm |
| duration | number | 持续时间 | 秒 |
| detectionScore | number | 检测置信度 | 0~100 |

## STM32数据包格式

ESP32接收STM32的三种数据包：

### 类型1：传感器数据 (0x01)

```
0xAA 0x01 [length] [temp] [humi] [light_h] [light_l] ... [checksum] 0x55
```

- temp: 温度（°C）
- humi: 湿度（%）
- light: 光照强度（16位）

### 类型2：设备状态 (0x02)

```
0xAA 0x02 [length] [led] [fan] [occupied] ... [checksum] 0x55
```

- occupied: 座位占用状态（0/1）

### 类型3：扩展传感器 (0x03)

```
0xAA 0x03 [length] [p1] [p2] [dist_h] [dist_l] [pir] ... [checksum] 0x55
```

- p1/p2: 压力传感器1/2（0/1）
- dist: 超声波距离（16位，cm）
- pir: PIR红外状态（0/1）

## 检测置信度计算

ESP32会自动计算检测置信度：

```cpp
int score = 0;
if (pressure1 || pressure2) score += 70;  // 压力传感器
if (distance > 0 && distance < 50) score += 25;  // 超声波
if (pirState) score += 5;  // PIR红外
```

- 总分 ≥ 70：判定为占用
- 总分 < 70：判定为空闲

- **状态变化**：立即上传
- **心跳包**：每10秒发送一次

## 故障排查

### 1. 连接失败

**症状**：`[COZE] ✗ Connection failed`

**解决方案**：
- 检查WiFi连接
- 确认域名正确：smkcc339ny.coze.site
- 检查网络是否能访问外网
- 尝试ping域名：`ping smkcc339ny.coze.site`

### 2. 数据未显示

**症状**：连接成功但网页无数据

**解决方案**：
- 检查deviceKey是否正确
- 查看ESP32串口输出的响应
- 确认STM32正在发送数据
- 检查数据包格式是否正确

### 3. WiFi频繁断开

**症状**：`[WIFI] Reconnecting...`

**解决方案**：
- 检查WiFi信号强度
- 确认WiFi密码正确
- 尝试更换WiFi频段（2.4GHz）
- 检查路由器设置

## 注意事项

1. **设备Key唯一性**：每个设备应该有唯一的deviceKey
2. **数据频率**：不要设置过高的上传频率，避免服务器压力
3. **网络稳定性**：确保WiFi连接稳定
4. **数据完整性**：确保STM32发送完整的数据包
5. **时区问题**：服务器使用UTC时间
