# 座位检测系统 - Web界面使用说明

## 系统架构

```
STM32 (座位检测) 
    ↓ 串口(UART3)
ESP32 (Web服务器 + WiFi)
    ↓ HTTP/WiFi
Web浏览器 (实时监控界面)
```

## 文件说明

### 1. ESP32_WebServer.ino
ESP32端的Web服务器代码，功能包括：
- 接收STM32通过串口发送的数据
- 提供Web服务器（端口80）
- 提供REST API接口
- 内嵌HTML界面

### 2. index.html
独立的Web前端页面，功能包括：
- 实时显示座位状态
- 显示温湿度、光照等传感器数据
- 显示系统信息
- 可配置ESP32 IP地址

## 使用步骤

### 方式一：使用ESP32内嵌界面（推荐）

1. **上传ESP32代码**
   - 打开Arduino IDE
   - 打开 `ESP32_WebServer.ino`
   - 修改WiFi配置：
     ```cpp
     const char* ssid = "你的WiFi名称";
     const char* password = "你的WiFi密码";
     ```
   - 上传到ESP32

2. **查看ESP32 IP地址**
   - 打开串口监视器（115200波特率）
   - 找到类似这样的输出：
     ```
     [WIFI] IP Address: 192.168.1.100
     ```

3. **访问Web界面**
   - 在浏览器中输入：`http://192.168.1.100`
   - 即可看到实时座位状态

### 方式二：使用独立HTML页面

1. **上传ESP32代码**（同方式一）

2. **打开HTML文件**
   - 直接双击 `index.html` 文件
   - 或者用浏览器打开

3. **配置ESP32 IP**
   - 在页面顶部输入ESP32的IP地址
   - 点击"连接设备"按钮
   - 等待连接成功

4. **查看实时数据**
   - 页面会每2秒自动刷新
   - 显示座位状态、传感器数据等

## 功能特性

### 实时监控
- ✅ 座位状态实时显示（占用/空闲）
- ✅ 温度、湿度、光照数据实时更新
- ✅ 状态持续时间统计

- ⚠️ 灰色 = 设备离线

- 最后更新时间

## 故障排查

### 问题1：无法连接ESP32

**检查项：**
1. ESP32是否已连接WiFi？
   - 查看串口输出确认
2. 电脑/手机是否与ESP32在同一WiFi网络？
3. IP地址是否正确？
4. 防火墙是否阻止了连接？

**解决方法：**
- 确保设备在同一局域网
- 尝试ping ESP32的IP地址
- 临时关闭防火墙测试

### 问题2：显示"设备离线"

**原因：**
- STM32未连接到ESP32
- 串口通信故障

**检查项：**
1. STM32是否正常运行？
2. 串口线是否连接正确？
   - ESP32 GPIO16 → STM32 PB10
   - ESP32 GPIO17 → STM32 PB11
   - GND → GND
3. 波特率是否匹配（115200）？

### 问题3：数据不更新

**检查项：**
1. 查看浏览器控制台是否有错误
2. 检查ESP32串口输出
3. 确认STM32正在发送数据

## 开发说明

### 修改刷新间隔
在HTML中找到：
```javascript
setInterval(updateStatus, 2000);  // 2000ms = 2秒
```
修改数值即可改变刷新间隔

### 添加新的传感器数据
1. 在ESP32代码中解析新数据
2. 在HTML中添加显示元素
3. 在JavaScript中更新数据

### 自定义界面样式
修改HTML中的CSS部分即可自定义颜色、布局等

## 技术栈

- **后端**: ESP32 + Arduino + WebServer库
- **前端**: HTML5 + CSS3 + JavaScript (原生)
- **通信**: REST API (JSON格式)
- **协议**: HTTP

## 注意事项

1. ESP32和访问设备必须在同一WiFi网络
2. 建议使用Chrome、Firefox、Edge等现代浏览器
3. 移动设备也可以访问（响应式设计）
4. 数据超时时间为10秒，超时会显示离线
