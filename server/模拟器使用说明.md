# 数据模拟器使用说明

## 📋 概述

提供两个模拟器用于测试系统功能：
1. **simulator.js** - 单座位模拟器（推荐用于测试）
2. **multi_simulator.js** - 多座位模拟器（演示扩展功能）

---

## 🚀 使用方法

### 方法1：单座位模拟器（推荐）

**启动步骤：**

1. 确保服务器正在运行
```bash
node server_v2.js
```

2. 新开一个终端，启动模拟器
```bash
cd bishe/server
node simulator.js
```

**效果：**
- 每5秒自动生成并发送传感器数据
- 每10秒发送心跳
- 随机切换座位状态（占用/空闲）
- 模拟真实传感器数据变化

**输出示例：**
```

========================================

========================================
```

### 方法2：多座位模拟器（演示用）

**启动步骤：**

1. 确保服务器正在运行
```bash
node server_v2.js
```

2. 新开一个终端，启动多座位模拟器
```bash
cd bishe/server
node multi_simulator.js
```

**效果：**
- 模拟5个座位的数据
- 每5秒随机选择一个座位发送数据
- 每30秒显示所有座位状态

## 📊 模拟数据说明

### 传感器数据范围

**占用状态：**
- 温度：26-28°C
- 湿度：65-75%
- 光照：30-50%
- 压力：1（有压力）
- 距离：30-50cm
- 声音：30%概率有声音
- PIR：20%概率检测到运动

**空闲状态：**
- 温度：24-26°C
- 湿度：55-65%
- 光照：70-90%
- 压力：0（无压力）
- 距离：120-170cm
- 声音：0（无声音）
- PIR：0（无运动）

### 状态切换

- 每次更新有10%概率切换状态（单座位）
- 每次更新有5%概率切换状态（多座位）
- 切换时持续时间重置为0

## 🔧 自定义配置

### 修改更新频率

编辑 `simulator.js`：

```javascript
// 修改数据发送间隔（毫秒）
setInterval(() => {
    this.sendData();
}, 5000);  // 改为 3000 = 3秒

// 修改心跳间隔
setInterval(() => {
    this.sendHeartbeat();
}, 10000);  // 改为 5000 = 5秒
```

### 修改状态切换概率

```javascript
// 修改状态切换概率
if (Math.random() < 0.1) {  // 改为 0.2 = 20%概率
    this.occupied = !this.occupied;
    ...
}
```

### 修改传感器数据范围

```javascript
// 修改温度范围
this.temperature = 26 + Math.random() * 2;  // 改为 25 + Math.random() * 5
```

## 📝 注意事项

1. **服务器必须先启动**
   - 模拟器依赖服务器API
   - 确保服务器在 http://localhost:3000 运行

2. **端口冲突**
   - 如果修改了服务器端口
   - 需要同步修改模拟器中的 SERVER_URL

3. **数据持久化**
   - 模拟器生成的数据会保存到 history.json
   - 测试完成后可以清空历史数据

4. **多座位限制**
   - 当前服务器只支持单座位
   - 多座位模拟器仅用于演示
   - 实际使用需要扩展服务器API
