# 云平台数据格式说明

本文档说明座位检测系统上传到云平台（如扣子平台）的数据格式。

## 数据上传方式

设备通过HTTP POST请求上传JSON格式数据到云平台API。

## 核心数据字段

### 1. 基础数据（必需）

```json
{
  "deviceId": "seat-001",
  "deviceName": "座位001",
  "location": "图书馆-阅览室A-01号",
  "occupied": true,
  "timestamp": "2026-04-10T14:00:00Z"
}
```

**字段说明：**
- `deviceId` (string): 设备唯一标识符
- `deviceName` (string): 设备名称
- `location` (string): 设备位置
- `occupied` (boolean): 座位占用状态（true=占用，false=空闲）
- `timestamp` (string): 数据采集时间（ISO 8601格式）

### 2. 传感器数据（推荐）

```json
{
  "temperature": 26,
  "humidity": 44,
  "lightLevel": 25605
}
```

**字段说明：**
- `temperature` (number): 温度，单位：摄氏度（°C），范围：-40~80
- `humidity` (number): 湿度，单位：百分比（%），范围：0~100
- `lightLevel` (number): 光照强度，ADC值，范围：0~65535

### 3. 检测详情（可选）

```json
{
  "pressure1": 1,
  "pressure2": 1,
  "distance": 15,
  "pir": 0,
  "detectionScore": 95
}
```

**字段说明：**
- `pressure1` (number): 压力传感器1状态（0=无压力，1=有压力）
- `pressure2` (number): 压力传感器2状态（0=无压力，1=有压力）
- `distance` (number): 超声波距离，单位：厘米（cm），范围：2~400
- `pir` (number): PIR传感器状态（0=无运动，1=检测到运动）
- `detectionScore` (number): 检测置信度，范围：0~100

### 4. 状态信息（可选）

```json
{
  "duration": 120,
  "occupiedCount": 5,
  "lastStateChange": "2026-04-10T13:58:00Z"
}
```

**字段说明：**
- `duration` (number): 当前状态持续时间，单位：秒
- `occupiedCount` (number): 今日占用次数
- `lastStateChange` (string): 最后一次状态变化时间

### 5. 设备信息（可选）

```json
{
  "wifiSignal": -55,
  "ipAddress": "10.150.235.123",
  "firmwareVersion": "1.0.0",
  "uptime": 3600
}
```

**字段说明：**
- `wifiSignal` (number): WiFi信号强度，单位：dBm，范围：-100~0
- `ipAddress` (string): 设备IP地址
- `firmwareVersion` (string): 固件版本号
- `uptime` (number): 设备运行时间，单位：秒

## 数据上传频率

- **正常模式**：每5秒上传一次
- **状态变化**：立即上传
- **心跳包**：每10秒发送一次（保持连接）

## HTTP请求格式

### 请求头

```
POST /api/seat/update HTTP/1.1
Host: your-cloud-platform.com
Content-Type: application/json
Authorization: Bearer YOUR_API_TOKEN
Content-Length: 256
```

### 请求体

```json
{
  "deviceId": "seat-001",
  "occupied": true,
  "temperature": 26,
  "humidity": 44,
  "lightLevel": 25605,
  "timestamp": "2026-04-10T14:00:00Z"
}
```

### 响应格式

**成功响应（200 OK）：**
```json
{
  "success": true,
  "message": "数据接收成功",
  "timestamp": "2026-04-10T14:00:01Z"
}
```

**错误响应（400 Bad Request）：**
```json
{
  "success": false,
  "error": "Invalid data format",
  "message": "缺少必需字段：deviceId"
}
```

    StaticJsonDocument<512> doc;
    doc["deviceId"] = "seat-001";
    doc["deviceName"] = "座位001";
    doc["location"] = "图书馆-A区-01号";

    doc["temperature"] = seatData.temperature;
    doc["humidity"] = seatData.humidity;
    doc["lightLevel"] = seatData.lightLevel;
    doc["timestamp"] = getISOTimestamp();
    
    String jsonData;
    serializeJson(doc, jsonData);
    
    int httpCode = http.POST(jsonData);

### 2. 添加时间戳函数

```cpp
#include <time.h>

String getISOTimestamp() {
    time_t now;
    struct tm timeinfo;
    char buffer[30];
    
    time(&now);
    gmtime_r(&now, &timeinfo);
    strftime(buffer, sizeof(buffer), "%Y-%m-%dT%H:%M:%SZ", &timeinfo);
    
    return String(buffer);
}

void setup() {
    // ... 其他初始化代码
    
    // 配置NTP时间同步
    configTime(0, 0, "pool.ntp.org", "time.nist.gov");
    Serial.println("[TIME] Waiting for NTP sync...");
    
    while (!time(nullptr)) {
        delay(1000);
        Serial.print(".");
    }
    Serial.println("\n[TIME] Time synchronized");
}
```

## 数据字段优先级

根据云平台需求，建议按以下优先级上传数据：

### P0（必需）
- deviceId
- occupied
- timestamp

### P1（强烈推荐）
- temperature
- humidity
- lightLevel
- location

### P2（推荐）
- duration
- detectionScore
- wifiSignal

### P3（可选）
- pressure1, pressure2
- distance
- pir
- occupiedCount
- 其他扩展字段
