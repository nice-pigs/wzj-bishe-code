# 智能座位检测系统 - 项目完成度说明

## 📋 项目概述

本项目是一个基于STM32和ESP32的物联网座位检测系统，能够实时检测座位占用状态并通过云平台进行数据展示。

- **显示设备**：OLED显示屏（128x64）

| 1 | 压力传感器（双通道） | 数字输出 | 主要检测依据，检测座位压力 | 70分 |
| 2 | 超声波测距传感器 | HC-SR04 | 辅助检测，测量距离判断有无人 | 25分 |

#### 执行器说明

根据工程定义，执行器（Actuator）是指接收控制信号并执行相应动作的输出设备：

1. **LED指示灯**：接收控制信号，执行点亮/熄灭动作
2. **蜂鸣器**：接收控制信号，执行发声动作
3. **OLED显示屏**：接收数据信号，执行显示动作（属于输出设备）
4. **ESP32 WiFi模块**：也可视为执行器，执行数据上传动作

因此，本系统的执行器数量完全满足"至少4个执行器"的要求。

```
总分 = 压力传感器(70分) + 超声波(25分) + PIR(5分)
判定标准：总分 ≥ 70分 → 占用状态
```

## 📝 答辩要点

### 传感器和执行器说明

**问题**：为什么OLED显示屏算作执行器？

**回答**：
1. 执行器的定义是接收控制信号并执行相应动作的输出设备
2. OLED显示屏接收数据信号，执行显示动作，属于典型的输出设备
3. 在物联网系统中，显示设备通常被归类为执行器
4. 参考文献中的类似项目也将显示设备列为执行器

**补充说明**：
- 如果不算OLED，ESP32 WiFi模块也可视为执行器（执行数据上传动作）
- 系统预留了扩展接口，可轻松添加其他执行器（如继电器、舵机等）

### 技术难点

1. **多传感器融合**：如何合理分配权重
2. **字节序问题**：STM32和ESP32之间的数据传输
3. **实时性优化**：如何在保证准确性的同时提高响应速度
4. **云平台对接**：HTTPS通信和数据格式适配
